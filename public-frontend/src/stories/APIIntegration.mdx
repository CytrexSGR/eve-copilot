# API Integration

## Service Layer

All API communication is centralized in `src/services/api/`. Each module exports typed functions that wrap Axios calls to a specific backend microservice. The API base URL uses relative paths (`/api/...`) which are proxied to the appropriate backend service via Vite dev server or nginx in production.

## TanStack React Query Configuration

API data is fetched and cached via TanStack React Query v5, configured in `src/lib/queryClient.ts`:

| Option | Value | Behavior |
|--------|-------|----------|
| `staleTime` | 60 seconds | Cached data is considered fresh for 60s; no re-fetch during this window |
| `gcTime` | 5 minutes | Unused cache entries are garbage-collected after 5 minutes |
| `retry` | 2 | Failed requests are retried twice before reporting an error |
| `refetchOnWindowFocus` | DEV only | Re-fetches stale data on tab focus in development, disabled in production |
| `refetchOnMount` | true | Always re-fetches on component mount if data is stale |

Components use `useQuery()` and `useMutation()` hooks for data operations, ensuring automatic caching, deduplication, and background refresh.

## Typical Data Flow

```
Component
  -> useQuery({ queryKey, queryFn: () => apiModule.getData() })
    -> Axios GET /api/domain/endpoint
      -> Vite proxy (dev) or nginx (prod)
        -> Backend microservice
          -> PostgreSQL / Redis / ESI
```
